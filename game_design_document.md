# ゲーム企画書

## 概要

* プラットフォーム：スマートフォンブラウザ（スマホ前提）
* 言語：JavaScript / HTML / CSS
* ホスティング：GitHub Pages（静的配信）
* ジャンル：脱出ゲーム風ナビゲーション → ミニゲーム → カードバトル

\---

## 画面構成と遷移

```
オープニング画面
├─ 「髪を切ってくれてありがとうございます。」→ ナビゲーション画面（ゲーム開始）
└─ 「ドコモに乗り換える」→ ドコモ公式サイト（外部リンク）

ナビゲーション画面
└─ キャラクターに話しかける → 数当てゲーム → カードバトル
       ├─ 負け → オープニング画面（ペナルティ演出付き）
       └─ 勝ち → 次のキャラへ（4人全員に勝利でエンディング）
```

## ナビゲーション画面

* 脱出ゲーム風のシンプルな画面切り替え
* プレイヤーキャラは画面中央に固定（移動しない）
* 東西南北4方向に各1枚の背景＋キャラクター画像を配置（画像生成AIで制作予定）

## 数当てゲーム

### ルール

1. キャラクターが「オレ美容師だけど、0000から9999の間で好きな数字教えて。」のような会話をする（セリフはキャラごとに異なる）
2. プレイヤーが4桁の数字（0000〜9999）を入力する
3. キャラごと毎回、内部でランダムな4桁の正解を生成しておく
4. **不一致の場合**：「違うよ○○○○だよ。」（正解の数字を告げる。セリフはキャラごとに異なる）
5. **完全一致の場合**：「正解。」（セリフはキャラごとに異なる）

キャラクターごとにゲームを行う1回限りの入力であるため、後述の ###カードの配布 もその都度行われる。

### スコアリング（桁ごとの一致判定）

各桁（1桁目〜4桁目）を独立して比較し、同じ位置で一致した桁の数をカウントする。

**例：**

* 正解「1234」、入力「4321」→ 一致0個
* 正解「1111」、入力「1100」→ 一致2個（1桁目・2桁目）

### 一致数に応じたカードの強さ

|一致桁数|カードのグレード|
|-|-|
|0個|グレード0（最弱）|
|1個|グレード1|
|2個|グレード2|
|3個|グレード3|
|4個|グレード4（最強）|


数当てゲームの一致桁数によって、付与されるカード群のグレードが変動する。
カードの数値（1から10）は、各グレード20枚の上下左右、合計「80か所の数値の平均」が特定の目標値になるように調整し、カードごとの個体差（例：全体的に弱いが上だけ数値が9の尖ったカードなど）の多様性を担保する。

相手（CPU）は全100枚のカードプールからランダムに5枚を手札にするため、相手のカード1方向あたりの数値の期待値は、全400か所の平均である「約5.5」となる。
プレイヤーはこの「相手の期待値5.5」に対して、数当てゲームの結果に応じたグレードのカードプールで挑むことになり、相対的な有利・不利が明確に変動する。

* 0個一致：グレード0（最弱）
80か所の数値の平均：3.0
相手の期待値（5.5）に対して圧倒的に不利な状態。
「うーわ、数当てゲームで全外しかよ...くっそ〜相手弱いカードきてくれ〜」という絶望感があるが、ごく稀に混ざっている一方向だけ強いカードの配置の妙や、相手の引きの弱さに賭けてワンチャンスを狙う。

* 1個一致：グレード1
80か所の数値の平均：4.2
相手の期待値に対してやや不利な状態。
「数当てゲームで1個当たったか。まあまあ、及第点だな。」という手応え。相手より少し手札が弱いが、プレイング（置く順番や場所の工夫）次第で十分にひっくり返せる、いわゆる「普通に戦える」難易度。

* 2個一致：グレード2
80か所の数値の平均：5.4
相手の期待値（5.5）とほぼ互角の状態。
「お、今回は数当てゲーム2個もあたったからこれいけるかも！？」とプレイヤーが一番熱くなれるフラットな勝負。運と実力が半々で絡むベストバランスの帯域。

* 3個一致：グレード3
80か所の数値の平均：6.8
相手の期待値を明確に上回る有利な状態。
「よっしゃ数当てゲーム3個も当たった！これは気を抜かなかったらいけるやろ！」という安心感がある。強力なカードが多く、セオリー通りに置いていけば順当に勝てる。

* 4個一致：グレード4（最強）
80か所の数値の平均：8.2
相手の期待値に対して圧倒的に有利な状態。
「え...奇跡の4個当てた。ラッキー。今回はさすがに勝ったかな？」というボーナスゲーム状態。9や10の数値が頻出し盤面を制圧できるが、一方向だけ極端に弱い隙（数値1など）を持たせることで、完全な思考停止による足元をすくわれるスリルは残しておく。

## カードシステム

### カードの構成

* カード総数：**100枚**（グレード0〜4 × 各20枚）
* 各カードのデータ：

  * カード名
  * 上・下・左・右 それぞれに **1〜10の数値**

### カードの配布

* 数当てゲームのグレードに対応したカード20枚の中からランダムに **5枚** プレイヤーへ付与
* 同一カードの重複はあり
* カードの中心にカード名、上下左右に数値

\---

## カードバトル

FF8のトリプルトライアドに準じたルール。お互いに手札はオープン。

### 基本ルール

* 盤面：3×3の9マス
* プレイヤー（青）と相手（赤）がそれぞれ5枚ずつカードを持つ
* **交互に1枚ずつ**カードを空きマスに配置（先攻・後攻で合計10手。先攻5手、後攻4手。）
* カードを置いたとき、上下左右に接した相手のカードと数値を比較：

  * 自分のカードの接触面の数値 **＞** 相手カードの接触面の数値 → **相手カードを自分の色（青）に変える**
  * 複数方向で同時に上回った場合、すべてのカードをひっくり返す
* 9マスが埋まったとき、**青と赤の枚数を比較して多い方が勝ち**
* 後攻の盤面に出していない手札もゲーム終了時の色の数に含める（後攻が不利なためアドバンテージ）

### 相手（赤）のカード

* 100枚のカード全体からランダムに5枚を選択

### 先攻・後攻の決定

* ランダムで決定
* キャラクターが決定に応じたセリフを喋る

  * 先攻の場合：「もちろんオレからな」
  * 後攻の場合：「これ後攻の方が強そうだからオレ後攻な」
  * ※ セリフはキャラごとに変えること

## 勝敗処理

### カードバトルで負けた場合

* キャラクターが「無理やり髪の毛伸ばしてやる！電脳世界へようこそ！！」と発言
* オープニング画面に戻る（進行状況はリセット）

### 4人全員に勝利した場合（エンディング）

1. 美容師キャラが「これローソンで使える500円クーポン券の番号○○○だけど」と発言
2. 続けて「こんなげーむにまじになっちゃってどうするの 完」と表示
3. ゲーム終了（それ以上の進展なし）

ちなみに、キャラクターに勝利すると、「お前の勝ちだ！」と言ったセリフがあるだけで再挑戦機能は無い。

## AIへ

- キャラクター名やセリフ、カード100枚は仮で自動生成してOKです。

### AI研究用要件

このゲームは最終的に機械学習AIを作るための題材とする。

そのため実装は以下の順序で行うこと!!

Phase1
ゲームルールをPythonで完全再現する。

実装対象

- 数当てゲーム
- カード生成
- カードバトル
- 勝敗判定

UI不要。

CLIで実行できればよい。

Phase2

ランダムAIを実装する。

行動はすべてランダム。

学習なし。

AI同士を10000戦以上実行できるようにする。

Phase3

特徴量を出力できるようにする。

例

- 手札
- 盤面状態
- 残り手札
- 先攻後攻
- 勝敗

をCSVへ保存する。

Phase4

機械学習モデルを作成する。

入力

- 盤面状態
- 手札

出力

- 次に置くカード
- 次に置く場所

まずは教師あり学習で実装する。

Phase5

自己対戦を行う。

学習済みAIとランダムAIを対戦させる。

勝率を計測する。

Phase6

JavaScript版へ移植する。

学習済みモデルを利用してCPU対戦を実装する。
